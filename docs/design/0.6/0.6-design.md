# ic-query 0.6 Design: Public Source Adapters

## Status

- Status: active implementation note
- Release line: `0.6.x`
- Decision style: public source-adapter API
- Primary implementation target: downstream crates that need typed reports
  without always using the built-in live host adapters

## Summary

0.6 opens the public source-adapter line. The 0.5 line opened request,
report, cache, refresh, and rendering APIs, but source traits stayed internal.
The 0.6 line exposes custom source boundaries incrementally, beginning with
the generic ICRC family and then moving through NNS and SNS families as their
source contracts are reviewed.

## Goals

1. Let host-only downstream crates provide custom source implementations for
   report builders without enabling `cli`.
2. Keep source APIs report-oriented and typed, not raw CLI argument or process
   boundaries.
3. Preserve explicit source endpoints in request/report DTOs so live network
   use remains visible.
4. Open source families one at a time, with public smoke tests for each family.

## 0.6.0 Slice

The first 0.6.0 slices expose generic ICRC, subnet catalog, and NNS registry
custom sources under `features = ["host"]`.

Generic ICRC:

- `IcrcSource`
- `LiveIcrcSource`
- source data DTOs such as `IcrcTokenData` and `IcrcTransactionsData`
- `build_icrc_*_report_with_source` builders

Subnet catalog:

- `SubnetCatalogSource`
- `SubnetCatalogSourceRequest`
- `LiveNnsRegistryRefreshSource`
- `load_or_refresh_subnet_catalog_with_source`
- `refresh_subnet_catalog_with_source`
- `build_subnet_catalog_list_report_with_source`
- `build_subnet_catalog_info_report_with_source`

NNS registry:

- `NnsRegistrySource`
- `NnsRegistrySourceRequest`
- `NnsRegistryVersionData`
- `LiveNnsRegistrySource`
- `build_nns_registry_version_report_with_source`

These keep the existing built-in report functions intact while allowing
downstream host tools to inject fixture, cached, proxy, or alternate transport
sources.

## 0.6.1 Slice

The 0.6.1 slice opens NNS inventory source adapters under `features =
["host"]`.

NNS inventory:

- `NnsNodeSource`
- `NnsDataCenterSource`
- `NnsNodeProviderSource`
- `NnsNodeOperatorSource`
- matching `*SourceRequest` DTOs
- matching `LiveNns*Source` adapters
- matching list/info `*_with_source` builders
- matching refresh `*_with_source` wrappers

These adapters use public list reports as their source return contracts rather
than exposing the lower-level registry inventory structs. That keeps registry
wire/projection details private while still letting downstream host tools
inject fixture, cached, proxy, or alternate transport inventory sources.

## 0.6.2 Slice

The 0.6.2 slice opens NNS proposal source adapters under `features =
["host"]`.

NNS proposals:

- `NnsProposalSource`
- `NnsProposalSourceRequest`
- `LiveNnsProposalSource`
- `build_nns_proposal_list_report_with_source`
- `build_nns_proposal_report_with_source`
- `refresh_nns_proposal_cache_with_source`

These adapters use public `NnsProposalRow` values as their source return
contract instead of exposing the lower-level governance candid wire DTOs. That
lets downstream host tools inject fixture, cached, proxy, or alternate
transport proposal sources for bounded list/detail reports and complete
snapshot refreshes.

## 0.6.3 Slice

The 0.6.3 slice opens the first SNS source adapters under `features =
["host"]`.

SNS list, info, token, and governance parameters:

- `SnsListSource`
- `SnsTokenSource`
- `SnsParamsSource`
- `SnsSourceRequest`
- `LiveSnsSource`
- source data DTOs for deployed SNS identity and token metadata:
  `MainnetSnsList`, `MainnetSns`, and `MainnetSnsToken`
- `build_sns_list_report_with_source`
- `build_sns_info_report_with_source`
- `build_sns_token_report_with_source`
- `build_sns_params_report_with_source`

The list and info adapters use the deployed SNS list as their source return
contract. Token and parameter adapters build on that same resolved SNS identity
before fetching per-SNS token metadata or governance parameters. This opens the
simple SNS report path for fixture, cached, proxy, or alternate transport
sources while leaving SNS proposals and neurons internal until their paged
snapshot contracts are reviewed separately.

## 0.6.4 Slice

The 0.6.4 slice extends SNS source adapters to governance proposal reports
under `features = ["host"]`.

SNS proposal detail, bounded proposal list, and complete proposal cache
refresh:

- `SnsProposalSource`
- `SnsProposalsSource`
- source data DTOs for proposal rows and pages: `MainnetSnsProposal`,
  `MainnetSnsProposals`, and `MainnetSnsProposalPage`
- `build_sns_proposal_report_with_source`
- `build_sns_proposals_report_with_source`
- `refresh_sns_proposals_cache_with_source`

These adapters use public `SnsProposalRow` values inside small source result
wrappers instead of exposing SNS governance candid wire DTOs. Bounded list
builders still apply the public request filters and sorts after source fetches,
and complete-cache refresh walks pages through the same injected source.

## 0.6.5 Slice

The 0.6.5 slice completes the SNS source-adapter path for neuron reports under
`features = ["host"]`.

SNS bounded neuron list and complete neuron cache refresh:

- `SnsNeuronsSource`
- source data DTOs for neuron rows and pages: `MainnetSnsNeurons`,
  `MainnetSnsNeuronPage`, and `SnsNeuronId`
- `build_sns_neurons_report_with_source`
- `refresh_sns_neurons_cache_with_source`

These adapters use public `SnsNeuronRow` values inside small source result
wrappers instead of exposing SNS governance candid wire DTOs. Bounded API-sort
reports fetch through the injected source, while cache-backed sorts continue to
read complete local snapshots. Complete-cache refresh walks neuron pages
through the same injected source.

## 0.6.6 Slice

The 0.6.6 slice opens NNS topology read-report source adapters under
`features = ["host"]`.

NNS topology summary, derived, and direct read reports:

- `NnsTopologySource`
- `NnsTopologySourceRequest`
- `LiveNnsTopologySource`
- `build_nns_topology_summary_report_with_source`
- `build_nns_topology_coverage_report_with_source`
- `build_nns_topology_versions_report_with_source`
- `build_nns_topology_health_report_with_source`
- `build_nns_topology_gaps_report_with_source`
- `build_nns_topology_capacity_report_with_source`
- `build_nns_topology_regions_report_with_source`
- `build_nns_topology_providers_report_with_source`

These adapters use complete public component list reports as the topology
source contract. That keeps topology joins and derived reports independent of
registry/cache implementation details while still letting downstream host tools
inject fixture, cached, proxy, or alternate transport topology snapshots.

## 0.6.7 Slice

The 0.6.7 slice completes NNS topology source adapters by opening topology
refresh under `features = ["host"]`.

NNS topology refresh:

- `NnsTopologyRefreshSource`
- `NnsTopologyRefreshSourceRequest`
- `refresh_nns_topology_report_with_source`

These adapters use complete public component refresh reports as the topology
refresh source contract. The built-in `refresh_nns_topology_report` path now
delegates through `LiveNnsTopologySource`, matching the read-report builders
and leaving component cache writes behind the existing component refresh APIs.
