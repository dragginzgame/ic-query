# 0.5 Changelog

Detailed patch notes for the `0.5.x` release line.

Root summary: [../../CHANGELOG.md](../../CHANGELOG.md)

When a patch introduces any new CLI surface, include a fenced `bash` command
example here and in the root changelog. New CLI surface includes commands,
subcommands, options, option values, and new supported option combinations.
Cleanup-only patches without CLI behavior changes should not include command
examples.

## 0.5.1 - 2026-06-25

### Added

- Adds a library feature-boundary CI check that verifies default and
  no-default `ic-query` builds, the wasm no-default target, and the
  `ic-query-cli` wrapper build.
- Adds negative dependency checks proving default and no-default `ic-query`
  builds do not include `clap`, `ic-agent`, Tokio, or `futures`; the
  no-default check also runs on `wasm32-unknown-unknown`.

### Fixed

- Corrects the release ledger after the library-boundary cleanup was published
  as `0.5.0` instead of a `0.4.x` patch.
- Adds the detailed `0.5.x` changelog file and moves the `0.5.0`
  library-boundary notes into it.
- Updates the README library dependency example from the `0.4` release series
  to the `0.5` release series.

## 0.5.0 - 2026-06-25

### Changed

- Moves the top-level process CLI runner from `ic-query` into `ic-query-cli`,
  leaving the library package focused on report APIs and family-level command
  adapters behind the `cli` feature.
- Removes now-unused library-owned top-level CLI help and command-family
  dispatch helpers after the runner move.
- Changes the `ic-query` default feature set to empty. Native host adapters and
  family-level CLI adapters now require explicit `host` or `cli` feature opt-in.
