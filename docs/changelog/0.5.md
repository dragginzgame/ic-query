# 0.5 Changelog

Detailed patch notes for the `0.5.x` release line.

Root summary: [../../CHANGELOG.md](../../CHANGELOG.md)

When a patch introduces any new CLI surface, include a fenced `bash` command
example here and in the root changelog. New CLI surface includes commands,
subcommands, options, option values, and new supported option combinations.
Cleanup-only patches without CLI behavior changes should not include command
examples.

## Unreleased

## 0.5.5 - 2026-06-26

### Changed

- Exposes `ic_query::sns` SNS info and token request/report DTOs under
  `--no-default-features`.
- Exposes `sns_info_report_text` and `sns_token_report_text` without `host` by
  decoupling those renderers from host-only SNS text helpers.
- Keeps SNS info/token live builders and source calls behind the existing
  `host` feature.
- Extends the SNS no-default public API smoke test to construct and render SNS
  list, info, and token reports.

## 0.5.4 - 2026-06-26

### Changed

- Splits the feature-boundary dependency checks into pure no-default and
  host-without-cli assertions.
- Asserts `ic-query --features host --no-default-features` does not pull
  `clap`, while pure/default/no-default builds still avoid `clap`, `futures`,
  `ic-agent`, and Tokio.
- Clarifies in the README and crate docs that no-default support is a host/CLI
  dependency boundary, not a `no_std` promise.

## 0.5.3 - 2026-06-26

### Changed

- Exposes `ic_query::subnet_catalog` catalog model, JSON parse/render, and
  subnet/canister resolver APIs under `--no-default-features`.
- Keeps subnet-catalog host cache refresh, report construction, stale-time
  policy, and human text report internals behind the existing host feature.
- Adds a no-default subnet-catalog public API smoke test that parses a catalog
  JSON payload and resolves both subnet and canister principals.
- Extends the library feature-boundary CI check so ICRC, NNS, SNS, and
  subnet-catalog public API smoke tests all run under `--no-default-features`.

## 0.5.2 - 2026-06-26

### Changed

- Exposes `ic_query::nns::registry` request, report, and text-rendering DTOs
  for registry-version reports under `--no-default-features`.
- Exposes `ic_query::sns` request, report, row, sort, and text-rendering DTOs
  for deployed SNS list reports under `--no-default-features`.
- Keeps NNS/SNS live report builders, cache IO, command parsing, and process
  dispatch behind the existing host/CLI feature gates.
- Makes `ic-query --features host --no-default-features` compile without the
  CLI feature by gating private NNS command modules behind `cli`.
- Extends the library feature-boundary CI check so the host-only build and NNS
  registry/SNS list public API smoke tests run under `--no-default-features`.

## 0.5.1 - 2026-06-25

### Added

- Adds a library feature-boundary CI check that verifies default and
  no-default `ic-query` builds, the wasm no-default target, and the
  `ic-query-cli` wrapper build.
- Adds negative dependency checks proving default and no-default `ic-query`
  builds do not include `clap`, `ic-agent`, Tokio, or `futures`; the
  no-default check also runs on `wasm32-unknown-unknown`.

### Fixed

- Corrects the release ledger after the library-boundary cleanup was published
  as `0.5.0` instead of a `0.4.x` patch.
- Adds the detailed `0.5.x` changelog file and moves the `0.5.0`
  library-boundary notes into it.
- Updates the README library dependency example from the `0.4` release series
  to the `0.5` release series.

## 0.5.0 - 2026-06-25

### Changed

- Moves the top-level process CLI runner from `ic-query` into `ic-query-cli`,
  leaving the library package focused on report APIs and family-level command
  adapters behind the `cli` feature.
- Removes now-unused library-owned top-level CLI help and command-family
  dispatch helpers after the runner move.
- Changes the `ic-query` default feature set to empty. Native host adapters and
  family-level CLI adapters now require explicit `host` or `cli` feature opt-in.
