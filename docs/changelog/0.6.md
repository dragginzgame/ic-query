# 0.6 Changelog

Detailed patch notes for the `0.6.x` release line.

Root summary: [../../CHANGELOG.md](../../CHANGELOG.md)

When a patch introduces any new CLI surface, include a fenced `bash` command
example here and in the root changelog. New CLI surface includes commands,
subcommands, options, option values, and new supported option combinations.
Cleanup-only patches without CLI behavior changes should not include command
examples.

## Unreleased

## 0.6.6 - 2026-07-01

### Added

- Opens the NNS topology read-report source-adapter path by exposing
  `NnsTopologySource`, `NnsTopologySourceRequest`, `LiveNnsTopologySource`,
  and topology summary, coverage, versions, health, gaps, capacity, regions,
  and providers `*_with_source` builders under `features = ["host"]`.
- Keeps the topology source contract report-oriented by accepting complete
  subnet catalog, node, data-center, node-provider, and node-operator list
  reports instead of exposing lower-level registry fetch or cache details.
- Adds downstream-style public API smoke coverage proving a host-only
  downstream crate can implement a custom topology source and build all
  topology read reports without using the built-in live topology adapter.

## 0.6.5 - 2026-07-01

### Added

- Completes the SNS source-adapter path for neuron reports by exposing
  `SnsNeuronsSource`, neuron source result DTOs, `SnsNeuronId`,
  `build_sns_neurons_report_with_source`, and
  `refresh_sns_neurons_cache_with_source` under `features = ["host"]`.
- Keeps the public neuron source contract report-oriented by returning
  `SnsNeuronRow` wrappers instead of exposing the SNS governance candid wire
  DTOs used by the built-in live adapter.
- Adds downstream-style public API smoke coverage proving a host-only
  downstream crate can implement a custom SNS neuron source and build bounded
  neuron reports and complete-cache refresh reports without using the built-in
  live SNS adapter.

## 0.6.3 - 2026-07-01

### Added

- Opens the first SNS source-adapter path by exposing `SnsListSource`,
  `SnsTokenSource`, `SnsParamsSource`, `SnsSourceRequest`, `LiveSnsSource`,
  and the SNS list/info/token/params `*_with_source` builders under
  `features = ["host"]`.
- Exposes the source-layer SNS identity and token data DTOs needed by those
  adapters (`MainnetSnsList`, `MainnetSns`, and `MainnetSnsToken`) while
  keeping SNS proposal and neuron source traits internal for later review.
- Adds downstream-style public API smoke coverage proving a host-only
  downstream crate can implement custom SNS list, token, and governance
  parameter sources and build reports without using the built-in live SNS
  adapter.

## 0.6.1 - 2026-07-01

### Added

- Opens the NNS inventory source-adapter path by exposing
  `NnsNodeSource`, `NnsDataCenterSource`, `NnsNodeProviderSource`, and
  `NnsNodeOperatorSource` under `features = ["host"]`.
- Exposes matching inventory source request DTOs, live source adapters,
  list/info `*_with_source` report builders, and refresh
  `*_with_source` wrappers so host users can inject fixture, cached, proxy, or
  alternate transport sources without enabling `cli`.
- Adds downstream-style public API smoke coverage proving a host-only
  downstream crate can implement custom NNS inventory sources and build list,
  info, and refresh reports without using the built-in live NNS registry
  adapters.

## 0.6.0 - 2026-07-01

### Added

- Opens the first public source-adapter path by exposing the generic ICRC
  `IcrcSource` trait, its source data DTOs, and the
  `build_icrc_*_report_with_source` report builders under `features =
  ["host"]`.
- Opens the subnet catalog source-adapter path by exposing
  `SubnetCatalogSource`, `SubnetCatalogSourceRequest`,
  `LiveNnsRegistryRefreshSource`, `load_or_refresh_subnet_catalog_with_source`,
  `refresh_subnet_catalog_with_source`, and the subnet catalog list/info
  `*_with_source` report builders under `features = ["host"]`.
- Opens the NNS registry source-adapter path by exposing `NnsRegistrySource`,
  `NnsRegistrySourceRequest`, `NnsRegistryVersionData`,
  `LiveNnsRegistrySource`, and `build_nns_registry_version_report_with_source`
  under `features = ["host"]`.
- Adds downstream-style public API smoke coverage proving a host-only
  downstream crate can implement custom generic ICRC, subnet catalog, and NNS
  registry sources and build reports without using the built-in live adapters.
